# 电力设备智能巡检系统开发文档

## 项目目标

本项目在两周内完成一个最小可演示的电力设备智能巡检系统，打通从摄像头采集、Atlas 边缘推理、目标检测、仪表识别、后端存储、告警生成、智能建议到前端展示的完整链路。

开发原则：

- 算法边界清晰：通用目标检测、仪表专项识别、业务规则和大模型建议分开实现。
- 工程归属明确：每个模块都有清晰输入、输出、交付物和验收标准。
- 两周内可完成：优先完成最小完整链路，再补充准确率、稳定性和展示效果。

全组开发必须遵守：

- [数据契约与接口规范](./contracts.md)
- [工程规范与联调标准](./engineering-standards.md)

## 系统总体流程

```text
摄像头采集
   ↓
Atlas 边缘端读取视频
   ↓
目标检测模型
   ├─ 设备识别
   ├─ 故障检测
   ├─ 仪表定位
   └─ 环境异常检测
   ↓
仪表专项识别与故障规则判断
   ↓
后端接收并保存结果
   ↓
生成告警、巡检记录和智能建议
   ↓
前端可视化展示
```

## 模块划分

| 模块 | 负责人 | 核心问题 |
| --- | --- | --- |
| Atlas 边缘推理 | 成员1 | 模型如何在开发板上稳定运行 |
| 数据集与目标检测 | 成员2 | 画面中有什么、在哪里、是否有可见异常 |
| 仪表识别与业务规则 | 成员3 | 仪表显示多少、是否超限、风险多大 |
| 后端与智能建议接口 | 成员4 | 数据如何保存、查询、告警和调用大模型 |
| 前端可视化与测试 | 成员5 | 结果如何展示、系统是否可用 |

## 推荐最终功能范围

必做：

- 3 种左右电力设备识别；
- 2 种左右外观异常检测；
- 1 种固定数字仪表识别；
- 1 种环境异常检测；
- Atlas 实际运行模型；
- 后端保存巡检结果；
- 前端展示实时和历史结果；
- 生成巡检报告。

选做：

- 大模型维修建议；
- 指针仪表识别；
- 多种仪表自动适配。

明确不做：

- 多摄像头；
- 温湿度传感器；
- 复杂用户权限系统。
