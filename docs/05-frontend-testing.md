# 成员5：前端可视化、系统测试与答辩

## 开发目标

负责将各模块结果整合为用户可操作、可展示的巡检系统，并负责系统联调测试、演示材料和答辩准备。

成员5解决的问题：

```text
结果如何展示、系统是否可用、答辩时如何稳定演示。
```

## 职责范围

主负责：

- 页面原型；
- Dashboard；
- 实时巡检页面；
- 故障中心页面，合并展示智能建议；
- 报告中心；
- 端到端测试；
- 联调测试；
- 演示视频、截图和答辩材料。

协作内容：

- 接入成员4提供的 API；
- 使用成员1提供的实时检测数据；
- 展示成员2的检测结果；
- 展示成员3的仪表读数、告警规则和智能建议内容。
- 所有页面字段、枚举和错误处理遵守 [数据契约与接口规范](./contracts.md)。

## 页面规划

| 页面 | 主要内容 |
| --- | --- |
| Dashboard | 设备数、巡检数、故障数、告警数、摄像头状态、开发板状态、模型运行状态 |
| 实时巡检 | 摄像头画面、检测框、设备识别结果、仪表读数、单位、状态、实时故障和风险等级 |
| 故障中心 | 故障类别、风险等级、处理状态、原因分析、处理建议、安全提示 |
| 报告中心 | 巡检报告查看和导出 |

页面需要的数据接口和字段见 [数据契约与接口规范](./contracts.md) 的“前端页面数据契约”，具体接口示例见 [API 端点规范](./api-spec.md)。

Dashboard 最新高风险项使用 `latestHighRiskAlarm` 字段。故障中心优先展示后端聚合后的 `EventItem`，不要在前端自行从原始检测结果派生告警等级。
所有页面统一支持 `pageState`：`loading`、`ready`、`empty`、`stale`、`partial_error`、`error`。

## 输入

- 成员4提供的 API 文档；
- 后端测试数据；
- Atlas 上传的实时检测图片；
- 检测框、类别和置信度；
- 仪表读数；
- 故障和告警记录；
- 智能建议内容；
- 巡检报告链接。

## 输出

前端需要展示：

- 实时画面；
- 检测框；
- 设备识别结果；
- 设备类别、位置和运行状态；
- 故障类型；
- 风险等级；
- 仪表读数；
- 仪表单位和读数状态；
- 故障处理状态；
- 智能建议；
- 历史巡检记录；
- 报告导出入口；
- 系统运行状态。

## 任务清单

### 页面开发

- 设计整体导航结构；
- 完成 Dashboard；
- 完成实时巡检页面；
- 在实时巡检页面展示设备识别、设备状态和仪表读数；
- 完成故障中心页面；
- 在故障中心合并展示智能建议；
- 完成告警展示；
- 完成报告中心页面；
- 在 Dashboard 展示系统状态。

### 实时巡检展示

- 展示摄像头或后端推送图片；
- 绘制检测框；
- 显示类别和置信度；
- 显示设备类别、位置和状态；
- 显示仪表读数；
- 显示仪表单位和读数状态；
- 显示故障和风险等级；
- 显示边缘端 FPS 和延迟；
- 显示关键帧标识、上传原因和结果状态；
- `resultStatus: no_frame` 时展示等待首帧状态；
- `resultStatus: stale` 时展示旧数据并提示实时结果已过期；
- 后端数据不可用时显示明确状态。

### 数据查询

- 接入设备列表接口；
- 接入巡检记录接口；
- 接入故障接口；
- 接入告警接口；
- 接入聚合事件接口；
- 接入 Dashboard 统计接口；
- 接入系统状态接口；
- 接入报告接口；
- 接入智能建议接口，并在故障中心合并展示。

### 测试与演示

- 编写端到端测试用例；
- 负责系统联调；
- 记录和跟踪缺陷；
- 准备模拟数据；
- 准备固定演示场景：正常无故障、仪表预警、严重故障、OCR 失败、系统降级无新帧；
- 准备备用图片和视频；
- 准备演示截图；
- 准备演示视频；
- 准备用户手册；
- 准备答辩 PPT。

## 验收标准

- Dashboard 能展示关键统计；
- Dashboard 能展示摄像头、开发板和模型运行状态；
- 实时巡检页面能展示图片、检测框、设备识别结果、设备状态和仪表读数；
- 实时巡检页面能展示仪表读数状态和实时风险等级；
- 能在故障中心查看故障、告警和智能建议；
- 能在故障中心查看聚合事件、出现次数、最佳证据帧和建议状态；
- 能查看或导出巡检报告，并展示生成中、失败和可下载状态；
- 后端接口异常时页面不会空白崩溃；
- 有完整演示流程和备用演示数据；
- 有覆盖关键帧展示、无帧空态、结果过期、建议降级和报告导出的端到端测试。
